# AddNodeBtnComponent 业务文档

## 1. 业务目的
提供流程节点添加入口，允许用户在审批流程图中从nodes目录选择并添加新的审批节点，并自包含完整的节点选择和添加交互流程。

## 2. 核心功能
- 显示添加节点的可视化圆形按钮
- 响应用户点击事件显示节点选择模态框
- 自包含节点选择模态框的显示和隐藏
- 从nodes目录获取并展示所有可用的审批节点（按序排列）
- 支持用户点击节点直接添加（无需额外确认步骤）
- 提供节点选择的视觉反馈（悬停效果和操作图标）
- 支持模态框的关闭操作
- **点击 add 节点后，自动添加审批节点和新的 add 节点**

## 3. 输入/输出
### 输入
- 无直接输入参数，组件内部维护审批节点列表

### 输出
- 通过 X6FlowGraph 服务向流程图添加新节点
- 自动建立节点间的连接关系
- 自动调整节点布局

## 4. 主要流程
1. 组件初始化时加载nodes目录下的所有审批节点信息
2. 渲染添加节点的圆形按钮
3. 用户点击按钮时显示节点选择模态框
4. 模态框中按序显示所有审批节点（并行审批、合并审批、普通审批）
5. 用户悬停节点时显示视觉反馈（边框高亮、图标放大）
6. 用户点击节点时执行节点添加逻辑：
   - 获取当前操作节点（通过 X6FlowGraph.currentNode）
   - 创建审批节点（普通审批/并行审批/合并审批）
   - 创建新的 add 节点
   - 更新节点关系：当前 add → 审批节点 → 新 add → 原下游节点
   - 删除旧连接线
   - 重建所有连接线
   - 调整节点布局
7. 关闭模态框

## 5. 依赖关系
### 服务
- X6FlowGraph：流程图管理服务，提供节点创建、连接线管理和布局调整功能

### 数据模型
- ApprovalNode：审批节点数据模型，包含类型、名称、描述、图标和形状信息
- NodeInfo：节点信息接口，包含节点类型、当前节点、前置节点和后置节点
- NodeType：节点类型枚举（Start、End、OperationNode、Info、ParallelApproveNode）

### UI组件
- ng-zorro-antd：按钮(nz-button)、图标(nz-icon)、模态框(nz-modal)和列表组件

### 性能优化
- 使用 ngFor 的 trackBy 函数提升列表渲染性能

## 6. 支持的审批节点类型
- **并行审批节点**：多个审批人同时审批，全部通过后流程继续
- **合并审批节点**：合并多个并行审批流程分支
- **普通审批节点**：单个审批人审批

## 7. 节点关系更新逻辑
### 7.1 普通审批节点
当用户点击 add 节点并选择普通审批节点时：
1. 保存当前 add 节点的原下游节点
2. 删除当前 add 节点到原下游节点的连接线
3. 更新当前 add 节点的 next 为审批节点
4. 更新审批节点的 prevs 为当前 add 节点
5. 更新审批节点的 next 为新 add 节点
6. 更新新 add 节点的 prevs 为审批节点
7. 更新新 add 节点的 next 为原下游节点
8. 更新原下游节点的 prevs，将当前 add 节点替换为新 add 节点

**最终节点关系**：当前 add → 审批节点 → 新 add → 原下游节点

### 7.2 并行审批节点
当用户点击 add 节点并选择并行审批节点时：
1. 保存当前 add 节点的原下游节点
2. 删除当前 add 节点到原下游节点的连接线
3. 创建并行审批节点、两个 add 节点（分支1和分支2）、合并审批节点
4. 更新当前 add 节点的 next 为并行审批节点
5. 更新并行审批节点的 prevs 为当前 add 节点
6. 更新并行审批节点的 next 为两个 add 节点（add1、add2）
7. 更新两个 add 节点的 prevs 为并行审批节点
8. 更新两个 add 节点的 next 为合并审批节点
9. 更新合并审批节点的 prevs 为两个 add 节点
10. 更新合并审批节点的 next 为原下游节点
11. 更新原下游节点的 prevs，将当前 add 节点替换为合并审批节点

**最终节点关系**：
```
当前 add → 并行审批节点 → add1 → 合并审批节点 → 原下游节点
                       → add2 →
```

**并行审批节点创建流程**：
- 创建并行审批节点（位于当前 add 节点下方）
- 创建第一个 add 节点（分支1，位于并行审批节点下方）
- 创建第二个 add 节点（分支2，位于并行审批节点下方）
- 创建合并审批节点（位于 add 节点下方）
- 更新所有节点的前后关系
- 重建连接线
- 调整节点布局
