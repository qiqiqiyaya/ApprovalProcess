# AddNodeBtnComponent 业务文档

## 1. 业务目的
提供流程节点添加入口，允许用户在审批流程图中从nodes目录选择并添加新的审批节点，并自包含完整的节点选择和添加交互流程。

## 2. 核心功能
- 显示添加节点的可视化圆形按钮
- 响应用户点击事件显示节点选择模态框
- 自包含节点选择模态框的显示和隐藏
- 从nodes目录获取并展示所有可用的审批节点
- 支持用户选择不同类型的审批节点
- 提供节点选择的视觉反馈和信息展示
- 支持节点添加的确认和取消操作

## 3. 输入/输出
### 输入
- 无直接输入参数，组件内部维护审批节点列表

### 输出
- 控制台日志：记录用户选择的节点信息
- 未来可扩展输出事件，用于向流程设计器传递节点添加请求

## 4. 主要流程
1. 组件初始化时加载nodes目录下的所有审批节点信息
2. 渲染添加节点的圆形按钮
3. 用户点击按钮时显示节点选择模态框
4. 用户在模态框中浏览并选择要添加的审批节点
5. 模态框显示选中节点的详细信息
6. 用户点击确认按钮时处理节点添加逻辑（当前仅记录日志）
7. 用户点击取消按钮或关闭模态框时取消操作
8. 模态框关闭后重置选择状态

## 5. 依赖关系
### UI组件
- ng-zorro-antd：按钮(nz-button)、图标(nz-icon)、模态框(nz-modal)和列表组件

### 数据模型
- ApprovalNode：审批节点数据模型，包含类型、名称、描述和图标信息

## 6. 支持的审批节点类型
- **并行审批节点**：多个审批人同时审批，全部通过后流程继续
- **合并审批节点**：合并多个并行审批流程分支
- **普通审批节点**：单个审批人审批

## 7. 使用示例
```html
<!-- 在流程设计器中使用添加节点按钮 -->
<app-add-node-btn></app-add-node-btn>
```

- 提供节点配置界面
