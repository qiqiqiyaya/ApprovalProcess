# 审批流程方式图解说明

本文档详细展示了17种常见的审批流程方式，通过图解形式帮助理解不同审批模式的逻辑和适用场景。

---

## 一、基础审批方式

### 1. 会签（Countersignature）

**定义：** 所有指定的审批人都必须审批通过，流程才能继续

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│      会签审批节点            │
├─────────┬─────────┬─────────┤
│ 审批人A │ 审批人B │ 审批人C │
└────┬────┴────┬────┴────┬────┘
     │         │         │
     │   全部通过   │
     │         │         │
     └────┬────┴────┬────┘
          │
          ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- ✅ 安全性最高
- ⏱️ 效率相对较低

**适用场景：** 重要决策、大额资金审批、合同签订等需要多方确认的场景

---

### 2. 或签（Or Sign）

**定义：** 在指定的审批人中，任意一人审批通过即可继续流程

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│      或签审批节点            │
├─────────┬─────────┬─────────┤
│ 审批人A │ 审批人B │ 审批人C │
└────┬────┴────┬────┴────┬────┘
     │         │         │
     │    任一通过   │
     │         │         │
     └────┬────┴────┬────┘
          │
          ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- ⚡ 效率高
- 🔄 灵活性强

**适用场景：** 日常事务性审批、替代性强的岗位（如主管请假，部门经理或副总经理都可审批）

---

### 3. 串签（Serial Sign）

**定义：** 审批人按照预设的顺序依次审批

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审批人A    │
└──────┬──────┘
       │ 通过
       ▼
┌─────────────┐
│  审批人B    │
└──────┬──────┘
       │ 通过
       ▼
┌─────────────┐
│  审批人C    │
└──────┬──────┘
       │ 通过
       ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- 📋 逻辑清晰
- 🎯 责任明确

**适用场景：** 需要逐级上报的审批（如：主管 → 经理 → 总监）

---

### 4. 并行审批（Parallel Approval）

**定义：** 多个审批人同时审批，互不影响

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│                           │
┌─────────┐  ┌─────────┐  ┌─────────┐
│ 审批人A │  │ 审批人B │  │ 审批人C │
└────┬────┘  └────┬────┘  └────┬────┘
     │            │            │
     │            │            │
     └────────┬───┴────────────┘
              │ 全部完成
              ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- ⏰ 节省时间
- ⚠️ 需要注意审批顺序的依赖关系

**适用场景：** 跨部门协同审批，各部门无先后依赖关系

---

## 二、高级审批方式

### 5. 会审（Joint Review）

**定义：** 多人共同讨论后，由主要负责人审批

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│         讨论环节              │
├─────────┬─────────┬─────────┤
│ 专家A   │ 专家B   │ 专家C   │
└────┬────┴────┬────┴────┬────┘
     │         │         │
     │   意见交流   │
     │         │         │
     └────┬────┴────┬────┘
          │
          ▼
┌─────────────────────────────┐
│      主要负责人审批          │
│         决策定案              │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- 💡 集思广益
- 🎓 专业性强

**适用场景：** 技术评审、方案论证等专业场景

---

### 6. 加签（Add Approver）

**定义：** 当前审批人可以临时增加其他审批人参与审批

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审批人A    │
└──────┬──────┘
       │
       │ 需要其他意见
       │
       ▼
┌─────────────────────────┐
│   临时增加审批人        │
├──────────┬──────────────┤
│ 审批人B  │  (新增)      │
└────┬─────┴──────┬───────┘
     │            │
     └───────┬────┘
             │
             ▼
┌─────────────┐
│  继续流程    │
└─────────────┘
```

**特点：**
- 🔧 灵活应对特殊情况

**适用场景：** 需要咨询其他部门或领导意见时

---

### 7. 减签（Remove Approver）

**定义：** 当前审批人可以移除后续部分审批节点

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审批人A    │
└──────┬──────┘
       │
       │ 加急情况
       │ 可跳过
       ▼
┌─────────────────────────────┐
│  原计划: 审批人B → 审批人C  │
│  实际:   ×跳过B×  → 审批人C │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- ⚡ 快速响应紧急情况

**适用场景：** 简化流程、加急审批（*通常需要权限控制*）

---

## 三、流转与处理方式

### 8. 转审批（Forward Approval）

**定义：** 当前审批人将审批权转交给他人

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审批人A    │
└──────┬──────┘
       │
       │ 无法处理
       │ (出差/请假)
       │
       ▼
┌─────────────────────────────┐
│      转交给审批人B           │
│    (一次性的临时转交)       │
└─────────────┬───────────────┘
              │
              ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- 🔄 解决临时不在岗问题

**适用场景：** 审批人出差、请假等无法处理时

---

### 9. 代理审批（Proxy Approval）

**定义：** 预设代理关系，原审批人不在时自动流转到代理人

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│  审批人A (主审批人)         │
│  ────────────────────       │
│  代理人: 审批人B            │
│  (预设的代理关系)           │
└──────┬──────────────────────┘
       │
       │ 主审批人不在时
       │ 自动流转
       ▼
┌─────────────────────────────┐
│     审批人B (代理人)         │
│      代为处理审批            │
└──────┬──────────────────────┘
       │
       ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- 🤖 自动化程度高

**适用场景：** 固定岗位的临时替代

---

### 10. 驳回（Reject）

**定义：** 审批不通过，流程终止或退回发起人修改

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审批人A    │
└──────┬──────┘
       │
       ├─────── 通过 ──────→ 流程完成
       │
       └─────── 驳回 ──────→ 流程终止
                       │
                       ▼
               ┌─────────────┐
               │ 需要重新申请 │
               └─────────────┘
```

**特点：**
- ❌ 明确否决

**适用场景：** 申请不符合要求或政策规定

---

### 11. 退回（Return）

**定义：** 退回到上一审批节点或发起人

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审批人A    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审批人B    │
└──────┬──────┘
       │
       ├─────── 通过 ──────→ 流程完成
       │
       └─────── 退回 ──────→ 审批人A (上一节点)
                       │
                       ▼
               修改后重新提交
```

**特点：**
- 🔄 允许修改后重新提交

**适用场景：** 需要补充材料或调整内容

---

### 12. 撤销/撤回（Withdraw）

**定义：** 发起人在流程结束前主动撤回申请

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       │ 主动撤回
       │
       ▼
┌─────────────┐
│  审批人A    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  审批人B    │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│   流程结束  │
└─────────────┘
       │
       │ ←── 流程中任意阶段
       │     发起人可撤回
       ▼
┌─────────────────────────────┐
│        申请已撤回             │
│    (可修改后重新提交)        │
└─────────────────────────────┘
```

**特点：**
- ✋ 给予发起人自主权

**适用场景：** 申请信息错误、情况发生变化

---

## 四、辅助与通知方式

### 13. 抄送（CC/Copy）

**定义：** 通知相关人员审批进度，但不参与决策

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│      审批流程节点            │
└──────┬──────────────────────┘
       │
       ├─────── 决策审批 ──────→ 审批人A
       │                           │
       └─────── 抄送通知 ──────→ 相关人员B
                                   (仅知情)
                                   (不参与决策)
       
       │
       ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- 🔔 信息透明

**适用场景：** 需要知情的领导、相关业务人员

---

### 14. 意见征集（Opinion Collection）

**定义：** 收集审批意见但不影响流程继续

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│        主流程审批            │
│        (不被阻塞)            │
└──────┬──────────────────────┘
       │
       ▼
┌─────────────────────────────┐
│     ─── 并行意见征集 ───     │
│                             │
│  ┌─────────┐   ┌─────────┐  │
│  │专家意见A│   │专家意见B│  │
│  │  (参考) │   │  (参考) │  │
│  └─────────┘   └─────────┘  │
└─────────────────────────────┘
       │
       ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- 📊 参考性强
- ▶️ 不阻塞流程

**适用场景：** 专家咨询、意见收集

---

## 五、组合与特殊方式

### 15. 混合审批（Hybrid Approval）

**定义：** 会签与或签的组合使用

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│       混合审批节点            │
│                             │
│  ┌──────────────────┐      │
│  │ 财务部门 (会签)   │      │
│  │  必须都通过       │      │
│  ├─────────┬─────────┤      │
│  │财务A    │ 财务B   │      │
│  └────┬────┴────┬────┘      │
│       └────┬───┘           │
└────────────┼────────────────┘
             │
             ▼
┌─────────────────────────────┐
│       混合审批节点            │
│                             │
│  ┌──────────────────┐      │
│  │ 法务部门 (或签)   │      │
│  │  任一通过即可     │      │
│  ├─────────┬─────────┤      │
│  │法务A    │ 法务B   │      │
│  └────┬────┴────┬────┘      │
│       └────┬───┘           │
└────────────┼────────────────┘
             │
             ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**示例：** 财务和法务必须都通过（会签），但法务部门任意一人审批即可（或签）

---

### 16. 条件审批（Conditional Approval）

**定义：** 根据业务数据自动选择审批人

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│      条件判断节点             │
│                             │
│   判断: 金额大小              │
└──────┬──────────────────────┘
       │
       ├──────────────────────┐
       │                      │
  ≤ 1万                    > 1万
       │                      │
       ▼                      ▼
┌─────────────┐      ┌─────────────┐
│  部门经理   │      │     总监     │
└──────┬──────┘      └──────┬──────┘
       │                    │
       │                    │
       └────────┬───────────┘
                │
                ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**示例：**
- 金额 ≤ 1万：部门经理审批
- 金额 > 1万：需总监审批

---

### 17. 百分比会签（Percentage Approval）

**定义：** 指定审批人中达到一定比例通过即可

**图解：**

```
┌─────────────┐
│   发起人    │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────┐
│     百分比会签审批节点        │
│     (需要 2/3 通过)          │
├─────────┬─────────┬─────────┤
│ 审批人A │ 审批人B │ 审批人C │
│   通过   │   通过   │   驳回   │
└────┬────┴────┬────┴────┬────┘
     │         │         │
     │   2/3 通过   │
     │   满足条件    │
     └────┬────┴────┬────┘
          │
          ▼
┌─────────────┐
│   流程完成  │
└─────────────┘
```

**特点：**
- 🗳️ 民主决策

**适用场景：** 委员会决策、投票类审批

## 总结对比表

| 审批方式 | 参与人数 | 决策逻辑 | 效率 | 安全性 | 适用复杂度 |
|---------|---------|---------|------|--------|-----------|
| 会签 | 多人 | 全部通过 | 低 | 高 | 高 |
| 或签 | 多人 | 任一通过 | 高 | 中 | 低 |
| 串签 | 多人 | 依次通过 | 中 | 高 | 中 |
| 并行 | 多人 | 同时审批 | 高 | 中 | 中 |
| 会审 | 多人 | 讨论后决策 | 中 | 高 | 高 |
| 加签 | 动态增加 | 临时新增 | 中 | 中 | 高 |
| 减签 | 动态减少 | 临时跳过 | 高 | 低 | 高 |
| 转审批 | 单人 | 转交他人 | 中 | 中 | 中 |
| 代理审批 | 单人 | 自动转交 | 高 | 中 | 中 |
| 驳回 | 单人 | 终止流程 | 低 | 高 | 低 |
| 退回 | 单人 | 回退修改 | 中 | 中 | 中 |
| 撤回 | 发起人 | 主动撤回 | 高 | 低 | 低 |
| 抄送 | 多人 | 仅通知 | 高 | 中 | 低 |
| 意见征集 | 多人 | 参考意见 | 高 | 中 | 中 |
| 混合审批 | 多人 | 组合逻辑 | 中 | 高 | 高 |
| 条件审批 | 动态 | 根据数据 | 高 | 高 | 高 |
| 百分比会签 | 多人 | 按比例 | 中 | 中 | 高 |

---

## 使用建议

1. **简单事务：** 优先使用或签、串签
2. **重要决策：** 使用会签、会审
3. **紧急情况：** 使用减签、转审批
4. **日常运营：** 使用代理审批、条件审批
5. **复杂业务：** 使用混合审批、百分比会签
